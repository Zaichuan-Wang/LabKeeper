# 实验室库存管理系统用户手册

本文档面向日常使用人员，说明如何登录系统、登记试剂和临床标本、查看库存空间、记录出库与移动，以及进行管理员维护。

## 1. 系统入口

### 1.1 打开系统

在浏览器中打开系统地址：

```text
http://127.0.0.1:5173/
```

如果是部署在服务器上的正式系统，请使用管理员提供的网址。

### 1.2 登录

在登录页输入：

- 用户名
- 密码

点击“登录”进入系统。

测试阶段页面可能会显示“测试管理员登录”按钮，用于内测快速进入系统。正式使用时应使用本人账号登录。

### 1.3 退出登录

进入系统后，右上角会显示当前用户信息。点击“退出”即可退出当前账号。

## 2. 用户角色

系统只保留两类角色：普通用户和管理员。

| 角色 | 主要权限 |
| --- | --- |
| 普通用户 | 查看工作台、空间可视化和流转记录；订购、到货、验证、上传验证图片、出库和修改个人密码 |
| 管理员 | 拥有全部功能，可管理用户、Excel、选项配置和删除空间 |

普通用户还可由管理员单独开启三个权限：

- 库存维护：新建/编辑试剂和临床标本、分装。
- 位置维护：移动库存、回滚移动、新建/编辑空间、调整空间父级和格位。
- 明细搜索：使用“库存空间 > 明细查询”的统一搜索。

如果页面上某些按钮不可点或看不到，通常是当前账号没有对应权限。

## 3. 页面总览

左侧导航包含以下入口：

| 页面 | 用途 |
| --- | --- |
| 工作台 | 查看库存概况、试剂分类、库存状态、空间结构和最近流转 |
| 登记入库 | 完成订购、到货、验证、临床标本入库和分装 |
| 流转记录 | 查看订购、到货、验证、出库和移动记录 |
| 库存空间 | 查看空间结构、查询库存明细、编辑库存、出库、移动库存 |
| 特殊关注 | 查询已过期、即将到期、未到货和待验证的对象 |
| 账号密码 | 修改当前账号密码 |
| 管理员 | 用户、Excel 和下拉选项维护，仅管理员可见 |

## 4. 日常使用流程

### 4.1 推荐流程

试剂和耗材推荐按以下顺序操作：

1. 在“登记入库 > 订购”记录订购信息。
2. 到货后在“登记入库 > 到货”关联订购记录，并填写有效期、数量和存放位置。
3. 需要验证时在“登记入库 > 验证”选择对应试剂，记录方法、结果和图片。
4. 日常使用中在“库存空间”查看位置、编辑信息、出库或移动。

临床标本推荐按以下顺序操作：

1. 在“登记入库 > 标本入库”登记样本号、样本类型、重量/体积、状态和存放位置。
2. 如需分装，在“登记入库 > 分装”选择来源库存并生成新管。
3. 在“库存空间”查看孔位或位置，必要时出库或移动。

## 5. 工作台

“工作台”用于快速了解库存状态，包括：

- 试剂/耗材、临床标本、空间节点等总量指标
- 试剂分类 Top 8
- 库存状态分布
- 空间结构摘要
- 最近流转记录

日常查看库存是否异常、近期是否有移动或出库时，可先从工作台进入。

## 6. 登记入库

“登记入库”包含五个标签页：订购、到货、验证、标本入库、分装。

### 6.1 订购

用于记录已提交或已确认的试剂/耗材订购信息。

填写内容包括：

- 名称
- 类型
- 品牌/厂家
- 货号
- 规格
- 数量
- 价格
- 用途/备注

点击“保存订购登记”后，记录会进入“流转记录 > 订购”。

如果需要重复订购已有试剂，可在“再次订购”区域搜索已有试剂，选择后点击“带入订购登记”，系统会自动填入部分信息，再根据本次订购情况修改数量、价格或备注。

### 6.2 到货

用于在试剂/耗材到货后生成库存记录。

操作步骤：

1. 选择“关联订购登记”。
2. 填写到货日期、到货数量和有效期。
3. 选择初始存放空间。
4. 如果存放在盒子中，选择盒内孔位。
5. 填写备注后点击“登记到货入库”。

如果暂时不知道具体位置，可以不指定存放空间，系统会默认放入“未归位”。

到货数量大于 1 时，可以选择“多件分别登记”。勾选时系统会生成多条库存记录，并按起始孔位连续放置；不勾选时只生成 1 条库存记录，库存数量记录为本次到货数量，只占 1 个位置或盒内孔位。

### 6.3 验证

用于记录试剂验证结果。

操作步骤：

1. 在“查找试剂”中输入名称、编号、货号、品牌或位置。
2. 选择“参考试剂”。
3. 填写或确认货号。
4. 选择验证日期、方法和结果。
5. 如有图片，上传验证图片。
6. 填写说明后点击“保存验证记录”。

验证图片支持 `png`、`jpg/jpeg`、`webp`、`tif/tiff`。系统会统一保存为压缩后的图片，便于后续查看和备份。

### 6.4 标本入库

用于登记临床标本。系统不记录个人身份信息。

填写内容包括：

- 样本号前缀和后缀，前缀可选择建议项，也可以手动输入或不填写；系统会拼接为业务样本号
- 冻存管数
- 样本类型
- 重量/体积
- 单位
- 状态
- 入库日期
- 存放空间
- 起始孔位
- 备注

保存后，系统会自动生成系统编号。样本号是业务编号，允许重复，例如同一个样本号可以分别登记血清和组织。如果一次录入多管，勾选“多件分别登记”时，系统会按冻存管数生成对应库存记录；不勾选时只生成 1 条库存记录，数量记录为冻存管数，只占 1 个位置或盒内孔位。选择盒子和起始孔位时，分别登记需要确认后续孔位有足够空位。

### 6.5 分装

用于从已有试剂/耗材或临床标本生成新的分装库存。

操作步骤：

1. 选择分装类型：临床标本或试剂/耗材。
2. 搜索来源库存。
3. 选择来源库存。
4. 填写新增管数。
5. 试剂/耗材可填写每管数量和单位。
6. 可点击“带入来源位置”快速使用来源库存当前所在空间；如来源有孔位，系统会把该孔位作为起点寻找空位。
7. 也可以手动选择存放空间和起始孔位。
8. 填写备注后点击“保存分装”。

分装后的新库存会拥有自己的库存记录，便于后续独立移动、出库和追溯。

## 7. 库存空间

“库存空间”是系统的核心页面，用于查看和管理实验室空间、盒内孔位、试剂/耗材和临床标本。

### 7.1 空间总览

空间总览左侧是空间导航，右侧显示当前空间。

空间维护里的结构类型只保留两类：

- 普通空间：用于表示实验室、冰箱、柜、液氮罐、架子、抽屉、托盘、分区等，可用行列框架容纳下级空间或直接库存。
- 盒子：末端空间，显示固定孔位，试剂和临床标本才会占用孔位。

实际日常称呼写在空间名称里，例如“-80℃ 冰箱 A”“液氮罐 1”“第三层抽屉”“冻存盒 A1”。盒内孔位会区分试剂、临床标本和空孔位。

点击空间名称可进入该空间。点击库存卡片或孔位可查看对应对象，并用于编辑、出库或移动。

### 7.2 “未归位”的含义

“未归位”是系统常驻位置，表示库存仍在库，但还没有指定到具体冰箱、架子、盒子或孔位。

以下情况通常会进入“未归位”：

- 到货时暂未填写存放空间
- 标本入库时暂未确定位置
- 分装时暂未指定位置
- 移动时选择清空具体空间

“未归位”不是报废或出库。已耗尽、已出库或不再占库存位置的对象，会释放位置并记录流转。

### 7.3 明细查询

“明细查询”用于按条件查找库存。

可筛选：

- 全部库存、试剂/耗材、临床标本、空间
- 关键词：名称、编号、货号、样本号、盒号、孔位、空间路径、备注
- 试剂类型、状态、验证状态
- 样本类型、状态
- 存放空间
- 是否包含下级空间

查询结果中可直接点击“编辑”“出库”。有库存维护权限的用户可编辑或出库；有位置维护权限的用户可移动库存。空间结果提供“打开空间”。

### 7.4 库存详情和时间线

在空间总览、盒内孔位或明细查询中点击“详情”，可以查看试剂或临床标本的当前信息和历史时间线。

时间线按时间倒序显示关键事件，例如：

- 创建、到货或标本入库。
- 位置移动、出库和移动回滚。
- 信息修改记录。
- 试剂相关的到货记录和“货号验证”记录。

“货号验证”表示该验证记录是针对试剂货号的参考结果，不代表每一支库存管都单独验证。点击时间线中的“验证详情”可以查看具体方法、结果、验证人和说明。

### 7.5 手动新建/编辑

用于直接维护试剂/耗材或临床标本。

试剂/耗材可填写：

- 名称
- 类型
- 品牌/厂家
- 货号
- 规格
- 数量和单位
- 状态
- 验证状态
- 存放空间和盒内孔位
- 有效期
- 备注

新建试剂/耗材时，数量大于 1 可选择是否“多件分别登记”。勾选时每件生成独立编号和独立库存位置；不勾选时只保存 1 条记录，数量字段记录多件数量，只占 1 个位置或盒内孔位。编辑已有记录时不会再拆分库存记录。

临床标本可填写：

- 系统编号
- 样本号
- 样本类型
- 重量/体积和单位
- 状态
- 入库日期
- 存放空间和盒内孔位
- 备注

如果要编辑已有记录，建议先在“空间总览”点选库存，或在“明细查询”中点击“编辑”。

### 7.6 出库登记

用于记录试剂/耗材或临床标本离开库存。

操作步骤：

1. 选择出库类型。
2. 输入关键词搜索对象。
3. 选择出库对象。
4. 填写原因和备注。
5. 点击“确认出库”。

出库后，系统会释放该对象占用的空间或孔位，并写入“流转记录 > 出库”。

### 7.7 库存移动

库存移动仅管理员可用。

操作步骤：

1. 选择库存类型：试剂或临床标本。
2. 通过关键词搜索库存对象。
3. 选择目标空间。
4. 如果目标是盒子，选择空孔位。
5. 填写原因和备注。
6. 点击“确认移动”。

系统会检查目标孔位是否已占用，避免覆盖已有库存。

### 7.8 空间维护

空间维护仅管理员可用，可在“空间总览”的“空间维护”菜单中进入。

可进行：

- 编辑当前空间
- 新建下级空间
- 新建顶层空间
- 删除当前空间

维护空间时可设置：

- 父级空间
- 名称
- 类型
- 位置码
- 盒子规格
- 框架行数和列数
- 备注

注意事项：

- “未归位”不能编辑、删除，也不能作为父级新建下级空间。
- 只有没有下级空间、没有库存的空间才能删除。
- 盒子按孔位管理；非盒子空间按统一框架管理。
- 行列都为 1 的非盒子空间会按无框架空间显示。

## 8. 流转记录

“流转记录”用于追溯库存历史，包括：

- 订购
- 到货
- 验证
- 出库
- 移动

当需要确认某个库存从哪里来、什么时候入库、是否验证、是否移动或出库时，可在这里查询。

移动记录中如支持回滚操作，回滚前系统会校验当前对象是否仍在该移动记录的新位置；如果目标位置已被占用，不能直接覆盖。

## 9. 特殊关注

“特殊关注”用于查看需要近期处理的试剂和流程状态。

操作方法：

1. 调整提醒天数，例如 30 天、60 天或 180 天。
2. 点击“查询”。
3. 查看“已过期”和“即将到期”列表。

建议定期查看该页面，及时处理即将过期或已经过期的试剂。

## 10. 账号密码

当前用户可在“账号密码”页面修改自己的密码。

操作步骤：

1. 输入旧密码。
2. 输入新密码。
3. 再次输入新密码。
4. 点击“更新密码”。

新密码至少 6 位。修改后请使用新密码重新登录。

## 11. 管理员功能

“管理员”页面仅管理员可见，包含 Excel、用户、选项、数据健康和数据库备份等标签页。

### 11.1 Excel 导出和模板

可选择数据表，并选择：

- 导出数据
- 下载模板

可设置行数上限。行数上限为 0 时，表示不限制导出行数。

### 11.2 Excel 导入

支持两种导入范围：

- 导入到单个表
- 按工作表名批量导入

支持两种写入模式：

- 新增行
- 按主键更新/插入

建议先下载模板，再按模板填写数据。导入前应确认目标表和写入模式，避免误更新已有数据。

数据库表导入属于高危维护入口，仅用于迁移、备份恢复或管理员修复数据。正式写入前系统会自动创建一次数据库备份，并在导入结果中显示备份文件名和完整性检查结果。

### 11.3 用户管理

管理员可新增和修改用户。

新增用户需要填写：

- 用户名
- 显示姓名
- 角色
- 是否启用账号

用户列表中可编辑用户信息，也可重置密码。当前系统重置密码后默认密码为：

```text
123456
```

重置后应提醒用户尽快登录并修改密码。

### 11.4 选项配置

选项配置用于维护常用下拉项，例如：

- 试剂类型
- 试剂状态
- 验证状态
- 验证方法
- 订购状态
- 样本类型
- 标本状态

修改后点击“保存选项配置”。后续登记和编辑表单会使用新的选项。

### 11.5 数据健康

“数据健康”是只读检查页，用于帮助管理员发现明显数据风险。

第一版会检查：

- 已耗尽试剂或标本仍占位置。
- 已订购试剂仍占位置。
- 库存指向不存在的空间。
- 同一空间同一孔位被多个对象占用。
- 同一来源标本下出现重复管号。
- 同一货号对应多个明显不同名称。

该页面只提示问题，不会自动修改数据。发现问题后，管理员应先确认实际库存和历史记录，再通过正常编辑、移动、出库或人工维护流程处理。

### 11.6 数据库备份

“数据库备份”用于管理员创建、下载和清理数据库快照。

可用功能包括：

- 手动创建备份，并填写备份原因。
- 下载最近备份文件。
- 删除单个备份文件。
- 设置定期备份间隔。
- 设置定期备份后的自动清理天数。
- 手动清理指定天数以前的过期备份。

备份会使用 SQLite 的一致备份能力，并执行完整性检查。删除和清理只会删除备份文件，不会影响当前运行数据库。

页面不提供网页一键恢复，避免误覆盖当前正式库。

恢复备份需要维护人员按部署文档停服务、替换数据库文件、删除旧 WAL/SHM 文件并重新检查系统。

## 12. 常见问题

### 12.1 找不到某个试剂或标本

可以尝试：

- 在“库存空间 > 明细查询”中选择“全部库存”
- 关键词使用名称、编号、货号、样本号或位置的一部分
- 勾选“包含下级”
- 检查是否被放入“未归位”
- 检查是否已经出库或耗尽

### 12.2 孔位无法选择

常见原因：

- 当前选择的空间不是盒子
- 目标孔位已被其他试剂或标本占用
- 选择的是非盒子空间，不能填写盒内孔位

### 12.3 到货或入库时还不知道位置

可以先不选择具体存放空间，系统会放入“未归位”。等确定位置后，由管理员在“库存空间”中移动到实际空间或孔位。

### 12.4 误把库存移动到错误位置

请在“流转记录 > 移动”中查找对应记录。如页面提供回滚操作，可按记录回滚。若目标位置已经被占用，需要先处理占用问题，再重新移动。

### 12.5 试剂已耗尽后还能看到历史吗

可以。试剂耗尽或数量归零后，系统只释放库存位置，不删除试剂行和验证记录。历史信息仍可追溯。

### 12.6 页面没有管理员入口

当前账号不是管理员。请联系系统管理员调整角色。

## 13. 数据填写建议

为了便于后续检索和追溯，建议：

- 名称使用实验室内部常用名称，避免只写缩写。
- 货号、品牌、规格尽量填写完整。
- 临床标本不要填写个人身份信息。
- 位置尽量精确到空间、盒子和孔位。
- 出库、移动、分装时填写简短原因。
- 验证结果和图片及时上传，减少后续查找成本。

## 14. 启动和关闭

本节供本机测试或内网部署维护人员使用。

项目中的 `dev_tools` 目录只用于部署前临时测试、演示库生成或一次性数据导入。系统正式部署后，日常维护应通过页面、后端正式接口或服务器计划任务完成，不应依赖 `dev_tools` 目录下的脚本。

### 14.1 本机启动

进入项目目录后运行启动脚本：

```bash
# Linux / macOS
./start.sh

# Windows PowerShell
.\start.ps1
```

可以自定义端口：

```bash
./start.sh --api-port 9000 --frontend-port 8080
```

启动后访问 `http://127.0.0.1:5173`（或你指定的前端端口）。

### 14.2 默认管理员

首次初始化时系统会创建默认管理员：

```text
admin / admin123
```

首次部署后必须尽快修改默认管理员密码。

### 14.3 数据备份

系统当前运行数据库为：

```text
db\lab_inventory.sqlite3
```

管理员可在“管理员 > 数据库备份”中创建一致快照，也可以开启定期备份，并按天数清理过期备份。进行批量导入、空间结构大调整或正式升级前，应先备份数据库。

正式环境的备份策略应由后端正式功能或服务器计划任务承担；不要把 `dev_tools` 目录下的临时脚本作为长期备份方案。
